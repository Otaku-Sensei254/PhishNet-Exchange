🔄 USER FLOWS SUPPORTED
═══════════════════════════════════════════════════════════════════════════════

FLOW 1: Admin Creates Team
   User A logs in
        ↓
   Goes to /dashboard
        ↓
   Sees "Create a New Team" form
        ↓
   Enters team name "My Team"
        ↓
   Clicks "Create Team"
        ↓
   ✅ Team created in PostgreSQL
   ✅ User A added as accepted member
   ✅ Team dashboard displays

FLOW 2: Admin Invites Member
   Admin enters member's email
        ↓
   System checks if email exists in PostgreSQL
        ↓
   If exists: Creates TeamMember (status=pending)
   If not: Shows "User not found" error
        ↓
   ✅ Invitation sent
   ✅ Success message shown

FLOW 3: Member Receives Invitation
   Member logs in
        ↓
   useTeamInvitations hook polls for pending invites
        ↓
   Finds pending invitation
        ↓
   ✅ TeamInvitationModal pops up automatically
   ✅ Shows team name and admin email
   ✅ Displays Accept/Reject buttons

FLOW 4: Member Accepts Invitation
   Member clicks "Accept" button
        ↓
   Backend updates TeamMember status to "accepted"
        ↓
   Modal closes
        ↓
   Member refreshes dashboard
        ↓
   ✅ Member added to team
   ✅ Can now see team dashboard
   ✅ Can view all team members

FLOW 5: Admin Removes Member
   Admin clicks "Remove" next to member
        ↓
   Confirms removal
        ↓
   Backend deletes TeamMember record
        ↓
   ✅ Member removed from team
   ✅ Member can still log in normally
   ✅ No longer sees team info

📊 DATABASE SCHEMA
═══════════════════════════════════════════════════════════════════════════════

User (EXISTING)
  ├── id: UUID
  ├── email: VARCHAR UNIQUE
  ├── password: VARCHAR
  ├── tier: ENUM (FREE|PRO|TEAM)
  └── ... other fields

Team (NEW)
  ├── id: UUID
  ├── teamName: VARCHAR
  ├── adminId: UUID (FK → User)
  ├── createdAt: TIMESTAMP
  └── updatedAt: TIMESTAMP

TeamMember (NEW)
  ├── id: UUID
  ├── teamId: UUID (FK → Team) [CASCADE DELETE]
  ├── userId: UUID (FK → User)
  ├── email: VARCHAR
  ├── status: VARCHAR (pending|accepted|rejected)
  ├── joinedAt: TIMESTAMP
  └── UNIQUE(teamId, userId)

🚀 API ENDPOINTS (8 Routes)
═══════════════════════════════════════════════════════════════════════════════

GET    /api/teams/check-user?email=user@example.com
       → {exists: true, userId, email}

POST   /api/teams/create
       → {message, team}

POST   /api/teams/add-member
       → {message, team, userExists}

POST   /api/teams/remove-member
       → {message, team}

POST   /api/teams/accept-invitation
       → {message, team}

POST   /api/teams/reject-invitation
       → {message}

GET    /api/teams/user/:userId
       → {userInTeam: boolean, team?}

GET    /api/teams/pending/:userId
       → {pendingInvitations: Team[], count}
